# Регистрация<a name="1"></a>
[Ссылка](https://pinspire.site/signup)

**Положительные тест-кейсы:**

1. При нажатии на кнопку **"Войти"** откроется окно авторизации, откуда можно вернуться обратно при нажатии на кнопку **"Регистрация"**.
2. При нажатии на кнопку в виде крестика в левом верхнем углу закроется окно регистрации.
3. При вводе любого набора символов в input поля **"Пароль"** или **"Повтор пароля"** текст отображается в виде точек.
4. При вводе в input поля **"Имя пользователя"** любого набора символов длиной от 4 до 50 символов, состоящего только из букв латинского алфавита, цифр и символов разделитей ("_", "-") и дальнейшем нажатии на кнопку **"Создать аккаунт"** input поле красным цветом выделяться не будет и сообщение с информацией под ним возникать также не будет.
5. При вводе в input поля **"Почта"** любого существующего почтового адреса и дальнейшем нажатии на кнопку **"Создать аккаунт"** input поле красным цветом выделяться не будет и сообщение с информацией под ним возникать также не будет.
6. При вводе в input поля **"Пароль"** любого набора символов длиной от 8 до 50 символов, состоящего только из букв латинского алфавита (причем имеющего хотя бы одну букву в верхнем регистре), цифр и специальных символов и дальнейшем нажатии на кнопку **"Создать аккаунт"** input поле красным цветом выделяться не будет и сообщение с информацией под ним возникать также не будет.
7. При вводе в input поле **"Повторите пароль"** тот же набор символов, что и в input поле "Пароль" по аналогичным требованиям
из пункта `6`, и дальнейшем нажатии на кнопку **"Создать аккаунт"** input поля **"Повторите пароль"** и **"Пароль"** цветом выделяться не будут и сообщений с информацией под ними также не будет.
8. При выполнении описанных выше пунктов `4`, `5`, `6`, `7` и дальнейшем нажатии на кнопку **"Создать аккаунт"** откроется главная страница с лентой.

**Негативные тест-кейсы:**

1. При нажатии на кнопку **"Создать аккаунт"** с пустыми input полями возникают сообщения под каждым из полей с информацией о том, что его нужно заполнить и сами input поля выделяются красным цветом.
2. При вводе эмодзи в любое input поле и дальнейшем нажатии на кнопку **"Создать аккаунт"** под каждым полем возникает сообщение о том, что оно содержит недопустимые символы и само input поле выделится красным цветом.
3. При вводе букв русского алфавита и специальных символов в input поле **"Имя пользователя"** и дальнейшем нажатии на кнопку **"Создать аккаунт"** под полем возникает сообщение о том, что оно содержит недопустимые символы и само input поле выделится красным цветом.
4. При вводе в input поле **"Почта"** набора символов, не содержащих "@" и дальнейшем нажатии на кнопку **"Создать аккаунт"** под полем возникает сообщение о том, что оно должено содержать символ "@" и само input поле выделится красным цветом.
5. При вводе в input поле **"Имя пользователя"** того имени, которое уже зарегистрировано и дальнейшем нажатии на кнопку **"Создать аккаунт"** над кнопкой возникает сообщение о том, что такой пользователь уже зарегистрирован.
6. При вводе в input поля **"Имя пользователя"** и **"Почта"** набора символов длиной до 4 символов и дальнейшем нажатии на кнопку **"Создать аккаунт"** под полями возникают сообщения с информацией о том, что они должны состоять из набора символов длиной более 4 символов и сами input поля выделятся красным цветом..
7. При вводе в input поля **"Имя пользователя"** и **"Почта"** набора символов длиной более 50 символов и дальнейшем нажатии на кнопку **"Создать аккаунт"** под полями возникают сообщения с информацией о том, что они должны состоять из набора символов длиной не более 50 символов и сами input поля выделятся красным цветом.
8. При вводе в input поля **"Пароль"** и **"Повтор пароля"** набора символов длиной от 8 до 50 символов, но не имеющего символа латинского алфавита в верхнем регистре и дальнейшем нажатии на кнопку **"Создать аккаунт"** под полями возникают сообщения с информацией о том, что поля должны содержать символ в верхнем регистре и сами input поля выделятся красным цветом.
9. При вводе в input поля **"Пароль"** и **"Повтор пароля"** различающихся наборов символов, удовлетворяющих пункту `6` из положительных тест-кейсов и дальнейшем нажатии на кнопку **"Создать аккаунт"** под полями возникают сообщения с информацией о том, что поля не совпадают и сами input поля выделятся красным цветом.
10. [**БАГ**] При вводе некорректных данных в input поля и нажатии на кнопку **"Создать аккаунт"** появляются сообщения под полями и сами поля выделяются красным цветом, но при повторном вводе данных сообщения не исчезают и цвет символов внутри полей остается красным (см. скриншот).

# Авторизация<a name="2"></a>
[Ссылка](https://pinspire.site/login)

**Положительные тест-кейсы:**

1. При вводе верного имени пользователя в input поле **"Имя пользователя"** , пароля в поле **"Пароль"** и дальнейшем нажатии на кнопку **"Войти"** происходит перенаправление на главную страницу с лентой, где в хэдере появляется аватарка пользователя.
2. При вводе в input поля **"Пароль"** любого набора символов длиной от 8 до 50 символов, состоящего только из букв латинского алфавита (причем имеющего хотя бы одну букву в верхнем регистре), цифр и специальных символов и дальнейшем нажатии на кнопку **"Войти"** input поле красным цветом выделяться не будет и сообщение с информацией под ним возникать также не будет.

**Негативные тест-кейсы:**

1. При нажатии на кнопку **"Войти"** с пустыми input полями возникают сообщения под каждым из полей с информацией о том, что его нужно заполнить и сами input поля выделяются красным цветом.
2. При вводе в input поле **"Пароль"** набора символов длиной от 8 до 50 символов, но не имеющего символа латинского алфавита в верхнем регистре и дальнейшем нажатии на кнопку **"Войти"** под полями возникают сообщения с информацией о том, что поле должно содержать символ в верхнем регистре и сами input поля выделятся красным цветом.
3. При вводе в input поле **"Пароль"** набора символов длиной до 4 символов и дальнейшем нажатии на кнопку **"Войти"** под полем возникает сообщение с информацией о том, что оно должно состоять из набора символов длиной более 4 символов и само input поле выделится красным цветом.
4. При вводе в input поле **"Пароль"** набора символов длиной более 50 символов и дальнейшем нажатии на кнопку **"Войти"** под полем возникает сообщение с информацией о том, что оно должно состоять из набора символов длиной не более 50 символов и само input поле выделится красным цветом.
5. При вводе в input поля **"Имя пользователя"** или **"Пароль"** неверных данных и дальнейшем нажатии на кнопку **"Войти"** над кнопкой возникает сообщение с информацией о том, что введено неверное имя пользователя или пароль и сами input поля выделятся красным цветом.
6. [**БАГ**] При вводе некорректных данных в input поле **"Пароль"** и нажатии на кнопку **"Войти"** появляется сообщение под полем и само поле выделяется красным цветом, но при повторном вводе данных сообщение не исчезает и цвет символов внутри поля остается красным (см. скриншот).

**Положительные тест-кейсы:**
